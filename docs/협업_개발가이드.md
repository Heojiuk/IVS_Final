# IVS V2V 군집주행 — 개발 협업 가이드

본인 **파트 행**을 찾아 해당 파일의 `★` 작업 구역을 구현하고, 상세 규격은 참조 문서를 따른다.

---

## 1. 프로젝트 구조

```
src/
  main.py          진입점 (실행은 이것만)
  messages.py      토픽 데이터 계약 (공용)
  core_module/     엔진 + 통신   — 통신/인프라
  algorithm/       인지 · 판단 · 모션
```

---

## 2. 파트별 RnR

| 파트        | 인원 | 담당자 | 파일                   | 입력 → [처리] → 출력                                                 |
|-------------|------|--------|------------------------|----------------------------------------------------------------------|
| 통신/인프라 | 1    | 지욱 | core_module/ · main.py | ego_state → [패킷·HMAC 송수신] → leader/follower_state · link_status |
| 인지        | 2    | 솔휘, 수 | perception.py          | 카메라·초음파 → [차선·객체·거리 인식] → scene                        |
| 판단        | 1    | 주성 | decision.py            | scene · link · leader_state → [행동·모드 결정] → command · mode      |
| 모션        | 2    | 정환, 승환 | motion_planning.py     | command · mode → [PWM 산출·조향·구동] → ego_state · GPIO             |

- 가운데 `[처리]` 가 각 파트가 구현할 핵심. 입력 토픽 read → 처리 → 출력 토픽 publish.
- 파일은 `algorithm/` 아래 (통신만 `core_module/`). 작업은 각 `step()` 의 `★` 구역, `bus.read`·`bus.publish` 는 변경 금지.
- 토픽 데이터 형식: `messages.py` (공용, writer 파트만 수정). 파트별 참조 문서는 아래 §3.

## 4. 시나리오별 RnR

각 셀 = 해당 상황에서 파트의 처리. 인지는 **데이터(거리·차선·신호)**, 판단은 **결정(behavior·mode)**, 모션은 **PWM 산출**.

| 상황 (조건) | 인지                                  | 판단                            | 모션                         | 통신                      |
|-------------|---------------------------------------|---------------------------------|------------------------------|---------------------------|
| 정상 추종   | 차선·전방 인식 → scene                | CRUISE 유지 (NORMAL)            | 차선중심 추종 steer+throttle | 자차 송신 / 상대 수신     |
| 곡선        | 차선 곡률·offset 산출                 | CRUISE 유지                     | offset·곡률로 steer 조향     | 정상 송·수신              |
| 전방 장애물 | 거리 dist_front_m 측정 (+front_clear) | dist_front_m 임계 → SLOW / STOP | throttle 감속 → 정지         | (후행) 선행 throttle 참고 |
| 정지선      | stop_signal 검출                      | STOP 후 재출발 판단             | 정지 후 재출발               | 정상 송·수신              |
| 차선 변경   | 양쪽 차선 추적                        | LANE_CHANGE + target_lane       | 목표 차선으로 steer 변경     | 정상 송·수신              |
| 링크 두절   | 정상 인지 지속                        | 끊김 감지 → DEGRADED(직전+서행) | mode 따라 서행               | link_status=LOST 발행     |
| 비상        | 위험 트리거 인식                      | ESTOP 발령                      | mode 따라 즉시 정지          | 상태 계속 송신            |

> 인지는 dist_front_m(거리값) 제공 → 판단이 임계값으로 SLOW/STOP 결정. (판단은 PWM 미참조, behavior로만 지시)
>
> Leader = 자율주행 / Follower = 선행차 추종 (leader_state 피드포워드).

## 5. 협업 규칙

- 모듈 간 데이터는 버스 경유만. 직접 호출 금지.
- `messages.py` 변경 시 사전 공유 — 모든 reader 파트에 영향.
- 버스 `read` 결과 `None` 가드 — 초기 주기에는 미발행 상태.
- `psk.key`(HMAC 키)는 git 제외. 양 차량에 동일 파일 배포.
